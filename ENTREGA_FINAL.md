# Checklist de Entrega Final - Care Daily (Sprint 4)

Este documento funciona como uma lista de verificação (checklist) para garantir que todas as exigências acadêmicas e técnicas da Sprint 4 da matéria **Testing, Compliance and Quality** tenham sido atendidas antes de submeter o trabalho ao portal do professor.

---

## 📋 Checklist de Validação de Entrega

* [ ] **Repositório GitHub público criado** (para disponibilização pública do código e documentação da Sprint 4).
* [ ] **Arquivos enviados na branch develop** (garantindo que o fluxo GitFlow do projeto seja respeitado).
* [ ] **Azure Boards criado** (projeto configurado no ambiente do Azure DevOps).
* [ ] **Testes manuais cadastrados no Azure Boards** (criação e organização dos 20 casos de testes planejados no board).
* [ ] **Dados de entrada preenchidos** (valores predefinidos de login, alimentos, calorias e medidas inseridos nos testes).
* [ ] **Dados de saída esperados preenchidos** (resultado final das ações definido nos testes).
* [ ] **Passos dos testes preenchidos** (lista ordenada de etapas de execução documentada nos testes).
* [ ] **Pelo menos 4 testes automatizados criados** (cenários TA001 a TA004 configurados no Selenium IDE).
* [ ] **Vídeo de execução gravado** (captura de tela demonstrando o Selenium IDE executando os testes automatizados com sucesso).
* [ ] **Link do GitHub preenchido** (referência no README.md raiz para o repositório público).
* [ ] **Link do Azure Boards preenchido** (referência no README.md raiz para o board público).
* [ ] **Link do vídeo preenchido** (link para assistir à gravação de execução dos testes automatizados).
* [ ] **Dados pessoais mascarados nas evidências** (garantia de conformidade com a LGPD, sem CPFs, e-mails ou telefones reais visíveis nos prints e vídeos).
