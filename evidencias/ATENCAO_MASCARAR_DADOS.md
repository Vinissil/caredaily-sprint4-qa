# Atenção: mascarar dados pessoais nas evidências

Antes de publicar o repositório ou enviar o ZIP, revise todos os prints da pasta `evidencias/`.

Devem ser mascarados, borrados ou substituídos por dados fictícios:

- CPF;
- telefone;
- e-mail;
- nomes reais;
- qualquer dado identificável de paciente, nutricionista ou administrador.

## Evidências que merecem atenção especial

- `print_admin_perfil_sprint4.png`
- `print_nutri_perfil_sprint4.png`
- `print_paciente_perfil_sprint4.png`
- `print_email_convite_sprint4.png`
- `print_nutri_convite_email_paciente_sprint4.png`
- `print_nutri_convite_paciente_sprint4.png`

Se os dados exibidos forem fictícios e próprios do ambiente de teste, mantenha essa informação descrita no README ou no relatório de compliance.
